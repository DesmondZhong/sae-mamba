Why This Matters

Sparse Autoencoders have revealed interpretable features inside Transformers. But Mamba processes information completely differently. Does the same technique still work?

The Key Question

Mamba compresses all past context into a fixed-size recurrent state. Can a Sparse Autoencoder still find clean, interpretable features in this compressed representation -- or is the information too entangled?

Monosemanticity Scores

Do the learned features represent single concepts? We measure automated monosemanticity scores (higher = more interpretable).

What is monosemanticity? ▼

A monosemantic feature responds to exactly one concept (e.g., "the word 'the'" or "mentions of sports"). A polysemantic feature fires for multiple unrelated concepts.

We compute monosemanticity by examining each feature's top-activating examples. If they cluster tightly around a single theme, the feature gets a high score. Scores near 0 indicate highly polysemantic or dead features; scores near 1 indicate clean, interpretable features.

Note: With only 20 features sampled per SAE, these scores should be interpreted as indicative rather than definitive.

Comparable Interpretability

Despite being denser, Mamba's early-layer features show monosemanticity scores competitive with Pythia. This suggests that SSM representations, while less sparse, still contain individually meaningful features.

Key Takeaways

What we learned from the first application of Sparse Autoencoders to State Space Models.

  1. SAEs work on Mamba

    Sparse Autoencoders successfully train on Mamba's residual stream with zero dead features and near-lossless reconstruction, establishing that mechanistic interpretability tools transfer to SSMs.

  2. Mamba is fundamentally denser

    Across all layers and L1 penalties, Mamba SAEs have 25-40% higher L0 than comparable Pythia SAEs. The recurrent state distributes information more uniformly.

  3. The Pareto frontier is shifted

    At every sparsity-reconstruction tradeoff point, Mamba requires more active features to achieve the same reconstruction quality. This is a fundamental architectural difference, not a training artifact.

  4. Features are still interpretable

    Despite higher density, individual Mamba features show comparable monosemanticity to Pythia features, especially in early layers.

  5. Zero feature overlap between architectures

    Cross-model feature matching yields zero overlap, suggesting Mamba and Transformers learn fundamentally different internal representations of language.

  6. Reconstruction is lossless

    Perplexity with SAE-reconstructed activations is indistinguishable from baseline for both architectures (ratio ~1.000x).

What's Next

Larger SAE Dictionaries

Scale to 16x and 64x expansion to find sparser Mamba features. Does higher expansion unlock interpretability?

SSM-Specific Extraction

Extract from Mamba's internal SSM state (A, B, C matrices) rather than the residual stream. Are there sparser representations hidden inside the recurrence?

Circuit Analysis

Use SAE features to trace circuits through Mamba. Can we identify "induction heads" or analogous mechanisms in SSMs?
