Phase 4: Induction-Circuit Localization in Mamba-1 2.8B

Where does a state space model do the "match-and-copy" computation that transformers do with attention heads?

← Main Dashboard (Phases 1-3) Method Mamba-1 localization C matrix Pythia control Mamba-2 Validation

Headline

Mamba-1's attention-free induction mechanism localizes to a single 16-dimensional slice (the C matrix of the selective-scan parameters) at layer 30. Patching it destroys

80% of induction signal

vs. Pythia-2.8B where the strongest single site reaches only

37% (distributed across ≥4 attention layers)

Mamba-1 concentrates a function that a transformer distributes — in a single 16-d subspace (state-space "readout" C), not in attention Q·K·V dot products.

1. Method

Induction-pair construction. Synthetic pairs

clean     = [prefix 8] P(8) [mid 32] P(8)     # pattern P repeats
corrupted = [prefix 8] P(8) [mid 32] P'(8)    # second pattern differs

Features-of-interest. Using the pre-existing Mamba-1 L32 SAE (TopK, K=64, 40,960 features), score every feature by (clean − corrupted) activation at the induction positions; take top-10.

Patching sweep. For every (layer, submodule), capture the submodule's activation on the corrupted run, patch into the clean run, measure patch_damage = 1 − (patched − corrupted) / (clean − corrupted). 1.0 = this submodule fully carries the induction signal.

Position-specific variant. Patch only at induction-pattern positions (48-55), or only at pre-induction positions (0-47), to separate "specifically induction" from "generic context."

Bug that mattered. HF's default MambaMixer.forward dispatches to cuda_kernels_forward, which calls causal_conv1d_fn(x, self.conv1d.weight, ...) directly — forward hooks on the Conv1d module never fire. Added force_slow_forward() helper (src/mamba_internals.py) that replaces each mixer's forward with slow_forward, which calls self.conv1d(x) and triggers hooks correctly.

3. The induction signal lives in C — a 16-dimensional slice

Mamba-1's x_proj is a Linear(d_inner=5120 → 192) that produces the selective-scan parameters:

  • time_step (Δ_pre): 160 dims — fed to dt_proj → discrete time-step Δ
  • B: 16 dims — state-space "input" matrix (how tokens write to state)
  • C: 16 dims — state-space "output" matrix (how state reads out to residual)

Patching each slice individually at L30:

Interpretation. Selective scan runs h_{t+1} = A(Δ) · h_t + B · x_t then y_t = C · h_t. Induction requires reading the state, not writing to it — so the "match found" signal is encoded in C. Earlier layers accumulate pattern memory into the state; at L30, C selectively reads out the matching component.

Necessary but not sufficient

The sufficiency direction tells a complementary story: patching clean C into an otherwise-corrupted run rescues only 5.6% of induction. L30's C is necessary (patching it destroys induction) but not sufficient (putting it back alone doesn't restore it). Induction requires both (i) the correct C readout AND (ii) the state built by layers 0–29 from the clean sequence.

5. Mamba-2 (SSD architecture) — the story does not transfer

Mamba-2 replaces selective scan with state-space duality (SSD) and merges all projections into a single in_proj that outputs [z, x, B, C, dt] (10,576 channels). We ran the analog of Phase B using the freshly-retrained Mamba-2 L32 SAE (see main report for the hook-fix that makes Mamba-2 SAE numbers comparable).

Mamba-2's induction signal is ~15× weaker than Mamba-1's at the same SAE

Mamba-1 (L32 SAE, plen=8)3.330.103.23
Mamba-2 (L32 SAE, plen=8)0.470.250.22

Pattern-length sweep on Mamba-2: gap grows from 0.16 (plen=4) to 0.27 (plen=32) but never approaches Mamba-1's 3.23. The weak signal isn't an artifact of short patterns.

Interpretation. The L30 x_proj.C localization is specific to selective scan, not to SSMs in general. Mamba-2's SSD, with its merged projections and smaller per-group d_state, distributes pattern-matching differently — consistent with external observations that SSD underperforms selective scan on in-context tasks at matched scale.

End-to-end picture of Mamba-1's induction

  1. Layers 0–29 accumulate pattern memory in the SSM state (distributed).
  2. At L30, ~10 sparse features in the d_inner=5120 pre-x_proj representation fire on pattern-match events (§6).
  3. Those features project through the 16 `C` rows of x_proj (§3).
  4. Selective scan y = C·h reads the matched pattern from state.
  5. The output propagates to L32, where a separate set of ~10 sparse features (identified by the L32 SAE) fire on induction-complete tokens.

Two SAEs, two different bases at two different sites of the same pipeline, both showing sparse coding of the induction signal.

7. Take-home

  • Localization: Mamba-1 2.8B implements its attention-free induction mechanism in a single 16-dim subspace (C matrix at L30 x_proj) of its 163,840-dim hidden state at that depth.
  • Mechanism: Earlier layers accumulate pattern memory into SSM state; at L30, C selectively reads it out. Δ and B slices carry nothing.
  • Necessary ≠ sufficient: Removing L30's C kills induction (80%); adding clean C into a corrupted run rescues only 5.6%. The mechanism is distributed across the state-building path (0-29) and concentrated at the readout.
  • Emergence: L26 → L28 → L30 transitions from patch_damage 0.00 → 0.19 → 0.83. Crosses 0.5 first at L30.
  • Robustness: Null-patch 0.00, Jaccard 1.0 across 5 seeds, 82% overlap across pattern lengths, features generalize to 5.4× natural-text activation ratio.
  • Contrast with Pythia: max single-site 0.37, distributed across 4+ attention layers. Mamba concentrates; transformers distribute. 2.3× concentration gap.
  • Mamba-2 differs: SSD's merged in_proj + smaller per-group state distributes pattern-matching more like a transformer than like Mamba-1; the "C matrix carries induction" property is specific to selective scan.
